The Data Life Cycle: From Ground to Guide

1. Trigger Event & Sensing Validation

A vehicle pulls into a spot. The **ground sensor** detects the magnetic change. A secondary ultrasonic reading validates the vehicle's presence, preventing false positives from metal debris or shadows.

Key Metric: False positive rate less than 0.1%.

2. Transmission & Aggregation

The **IoT sensor** sends a tiny payload (e.g., "P5: Occupied") to the nearest **Gateway**. The Gateway buffers the data and transmits it securely via a high-speed cellular network (4G/5G) to the Cloud API endpoint.

Technology Used: LoRaWAN (Long Range Wide Area Network).

3. Cloud Processing & API Update

The **Cloud Platform** ingests the data, timestamps it, and runs it through a proprietary algorithm to filter potential noise. The validated status is written to the **Fast-Access Database** and instantly updates our public API.

Benefit: Data is normalized and ready for consumption in under 500 milliseconds.

4. User and Operator Visualization

The **Live Map** page and the mobile application consume the public API data. Simultaneously, city operators view the data via the **Admin Dashboard** for performance monitoring and enforcement.

Outputs: Enforcement alerts, occupancy trends, and real-time guidance.
